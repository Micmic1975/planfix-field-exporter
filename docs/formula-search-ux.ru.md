# UX сценарий поиска использований поля в формулах

Выбранный вариант: **A — запуск поиска от выбранного поля в текущей таблице**.

## Цель

Пользователь выбирает пользовательское поле Planfix в текущей таблице и
запускает поиск: где это поле используется в формулах вычисляемых полей.

## Точка входа

В текущем окне рядом с кнопками результатов добавить кнопку:

```text
Обновить индекс
```

Она выполняет долгий read-only обход Planfix и сохраняет локальный индекс
формул.

Для поиска по уже созданному индексу используется кнопка:

```text
Найти в формулах
```

Также добавить пункт в контекстное меню таблицы:

```text
Найти использования этого поля
```

Оба действия запускают один и тот же сценарий.

## Условия доступности

Кнопка/действие должны работать только если:

- поля текущего источника загружены;
- пользователь выбрал строку поля в таблице;
- выбранная строка содержит `ID поля` и `Название поля`;
- настройки Planfix заполнены.

Если условие не выполнено, показывать русское сообщение:

```text
Сначала выберите поле в таблице.
```

## Что считается выбранным полем

Выбранное поле определяется по активной строке таблицы, а не по активной
ячейке.

Из строки нужны:

- `ID поля`;
- `Название поля`;
- `Тип источника`;
- `ID источника`;
- `Название источника`;
- `ID типа поля`;
- `Тип поля`.

Если нужные столбцы скрыты, приложение все равно должно брать данные из
`current_rows`, а не из видимых ячеек таблицы.

## Локальный индекс формул

Чтобы не выполнять долгий обход Planfix при каждом выбранном поле, приложение
хранит локальный индекс формул в выбранной пользователем папке выгрузок:

```text
<Папка выгрузок>/formula_index/
```

Минимальный набор файлов:

- `formula_index.json` — найденные формулы и разобранные ссылки `{{...}}`;
- `index_status.json` — статус последнего обновления;
- `index_errors.json` — ошибки отдельных справочников, если Planfix вернул
  ошибку.

`Найти в формулах` не обращается к Planfix API. Если индекс ещё не создан или
имеет неподдерживаемую версию, приложение сообщает пользователю, что нужно
нажать `Обновить индекс`.

При обновлении индекса проверяются:

- пользовательские поля задач;
- пользовательские поля контактов;
- текстовые поля записей справочников, если в них есть ссылки `{{...}}`;

Проекты, сотрудники, основные поля, справочники и аналитики остаются в модели,
но не являются активным источником формул до подтверждения через реальные
ответы API.

## Прогресс

Обновление индекса и поиск по индексу должны выполняться не в основном
Tkinter-потоке.

В status bar показывать:

- текущий этап;
- сколько элементов проверено и сколько осталось, если общее количество уже
  известно;
- время последнего запроса;
- путь текущего output-файла.

Минимальный пример:

```text
Индексация справочника: Предупреждения, проверено 40/120, последний запрос 17:29:38
```

## Окно результатов

Результаты открываются в отдельном окне.

Заголовок:

```text
Использования поля «<Название поля>»
```

В верхней части окна:

- выбранное поле: название и id;
- тип исходной сущности: задача, контакт, проект и т.п.;
- количество найденных совпадений;
- путь CSV/JSON output-файла, если сохранен.

Основная таблица результатов:

- `Уверенность`
- `Где найдено`
- `ID вычисляемого поля`
- `Название вычисляемого поля`
- `Справочник / запись / поле`, если совпадение найдено в текстовом поле
  справочника
- `Найденная ссылка`
- `Совпавший сегмент`
- `Формула`

Дополнительные данные сохраняются в CSV/JSON, даже если не все видны в таблице:

- тип источника формулы;
- id источника формулы, если есть;
- название источника формулы, если есть;
- id справочника;
- название справочника;
- ключ записи справочника;
- название записи справочника;
- id текстового поля справочника;
- название текстового поля справочника;
- `objectType`;
- id выбранного поля;
- название выбранного поля;
- разобранные сегменты ссылки;
- индекс совпавшего сегмента.

## Действия в окне результатов

Минимальные действия:

- `Экспорт CSV`;
- `Копировать формулу`;
- `Копировать ссылку`;
- `Закрыть`.

Если результатов нет, показывать:

```text
Использования поля в формулах не найдены.
```

## Уровни уверенности

В первой версии в таблице показывать только надежные совпадения:

- `exact_segment`;
- `exact_quoted_segment`;
- `casefold_segment`.

`raw_substring` не показывать по умолчанию, чтобы не вводить пользователя в
заблуждение. Его можно добавить позже как диагностический режим.

## Output-файлы

Каждый запуск поиска по индексу должен сохранять результаты в выбранной
пользователем папке выгрузок:

```text
<Папка выгрузок>/formula_search/<timestamp>/
```

Минимальный набор файлов:

- `selected_field.json`;
- `results.json`;
- `results.csv`;
- `progress.log` или эквивалентный журнал, если будет нужен.

JSON сохранять с `encoding="utf-8"` и `ensure_ascii=False`.
CSV сохранять как `utf-8-sig`.

## Поведение при ошибках

Если Planfix API или сеть вернули ошибку:

- обновление индекса пытается продолжить работу, если можно пропустить
  проблемный справочник;
- токен не показывается;
- пользователь видит русское сообщение;
- уже сохраненный частичный индекс остаётся в папке `formula_index`;
- подробности сохраняются в `index_errors.json`.

Если пользователь нажимает `Остановить`, приложение завершает текущий HTTP-запрос,
сохраняет частичный индекс и возвращает управление интерфейсу.

## Что не входит в первую версию UX

В первую версию не включаем:

- отдельную вкладку глобального поиска;
- выбор поля из общего списка всех полей Planfix;
- поиск по простой подстроке в сырой формуле;
- автоматическое исправление формул;
- массовое обновление полей Planfix.

## Итоговое решение

Реализуем поиск как действие над выбранной строкой текущей таблицы:

1. Пользователь загружает поля объекта или шаблона задачи.
2. При необходимости нажимает `Обновить индекс`.
3. Приложение выполняет фоновый read-only обход Planfix с паузой между
   запросами не менее `1.25` секунды.
4. Пользователь выбирает строку нужного поля.
5. Нажимает `Найти в формулах` или пункт контекстного меню.
6. Приложение выполняет локальный поиск по индексу без HTTP-запросов.
7. Результаты открываются в отдельном окне и сохраняются в CSV/JSON.
