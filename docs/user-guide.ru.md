# Краткое руководство пользователя

## Для чего нужна программа

`Planfix Field Exporter` помогает просматривать пользовательские поля Planfix и
искать, где выбранное поле используется в формулах.

Приложение работает только на чтение. Оно не меняет поля, справочники, записи,
объекты, контакты или шаблоны задач в Planfix.

## Что умеет приложение

- показывает поля объектов Planfix;
- показывает поля шаблонов задач Planfix;
- фильтрует и сортирует таблицу полей;
- позволяет выбирать видимые столбцы;
- экспортирует список полей в CSV;
- строит локальный индекс формул;
- ищет использования выбранного поля в формулах вычисляемых полей и текстовых
  полях записей справочников.

## Первый запуск

1. Запустите приложение.
2. В блоке `Подключение` укажите аккаунт Planfix без `.planfix.ru`.
   Например, для `https://engineering.planfix.ru` нужно ввести `engineering`.
3. Вставьте токен доступа без слова `Bearer`.
4. Выберите папку выгрузок, если папка по умолчанию не подходит.
5. Нажмите `Сохранить настройки`.

Аккаунт и папка выгрузок сохраняются в настройках приложения. Токен хранится в
системном хранилище секретов Windows.

## Просмотр полей

1. В блоке `Источник` выберите `Поля объекта` или `Поля шаблона задачи`.
2. В поле выбора источника начните вводить название или ID.
3. Выберите нужный объект или шаблон задачи.
4. Поля загрузятся автоматически.
5. В блоке `Столбцы` отметьте данные, которые нужно видеть в таблице.

В таблице можно фильтровать значения через строку фильтров под заголовками и
сортировать столбцы нажатием на заголовок.

## Настройка справочников для поиска формул

Формулы могут храниться не только в вычисляемых полях, но и в текстовых полях
записей справочников.

Чтобы исключить большие или ненужные справочники:

1. Нажмите `Справочники`.
2. Дождитесь загрузки списка справочников.
3. Выберите один или несколько справочников.
4. Нажмите `Переключить выбранные`.
5. Нажмите `Сохранить`.

После сохранения приложение показывает подтверждение в окне справочников.

## Обновление локального индекса формул

Перед быстрым поиском нужно создать локальный индекс формул.

1. Закройте другие экземпляры приложения и не запускайте параллельные скрипты,
   которые обращаются к Planfix API.
2. Нажмите `Обновить индекс`.
3. Дождитесь завершения.

Во время обновления приложение показывает текущий этап, количество обработанных
элементов, время последнего запроса и путь файла индекса.

Между любыми HTTP-запросами к Planfix приложение делает паузу не менее `1.25`
секунды. Поэтому первый полный индекс может строиться долго, особенно если в
аккаунте много записей справочников.

Индекс сохраняется в выбранной папке выгрузок:

```text
<Папка выгрузок>\formula_index
```

Если нужно остановить процесс, нажмите `Остановить`. Частично собранный индекс
будет сохранён.

## Поиск использований выбранного поля

1. Загрузите поля объекта или шаблона задачи.
2. Выберите строку нужного поля в таблице.
3. Нажмите `Найти в формулах`.

Обычный поиск использует локальный индекс и не делает HTTP-запросов к Planfix.
Если индекс ещё не создан, приложение попросит сначала нажать `Обновить индекс`.

Результаты открываются в отдельном окне. В таблице видны:

- где найдено использование;
- название вычисляемого поля;
- название справочника;
- ключ и название записи справочника;
- название текстового поля справочника;
- найденная ссылка `{{...}}`;
- совпавший сегмент.

Результаты сохраняются в:

```text
<Папка выгрузок>\formula_search\<дата_и_время>
```

Основной файл для работы в Excel:

```text
results.csv
```

## Экспорт списка полей

Кнопка `Экспорт CSV` сохраняет текущий список полей в выбранную папку выгрузок.
В CSV попадают загруженные строки и отмеченные в блоке `Столбцы` столбцы.

## Копирование данных из таблицы

Щёлкните по ячейке таблицы и используйте:

- `Ctrl+C` для копирования значения;
- контекстное меню правой кнопкой мыши;
- `Фильтровать по значению`, чтобы быстро применить фильтр по выбранной ячейке.

Ширину столбцов можно менять мышью за правую границу столбца.

## Где хранятся данные

- настройки приложения:
  `AppData\Roaming\PlanfixTools\Planfix Field Exporter\settings.json`;
- токен:
  системное хранилище секретов Windows;
- экспорт CSV:
  выбранная пользователем папка выгрузок;
- локальный индекс формул:
  `<Папка выгрузок>\formula_index`;
- результаты поиска по формулам:
  `<Папка выгрузок>\formula_search\<дата_и_время>`.

## Типовые ситуации

### Индекс строится слишком долго

Откройте `Справочники`, исключите большие справочники, где точно нет формул, и
запустите `Обновить индекс` заново.

### Planfix вернул ошибку по одному справочнику

Приложение старается продолжить обновление индекса и пропустить проблемный
справочник. Подробности сохраняются в `index_errors.json`.

### Поиск не нашёл поле, хотя оно есть в Planfix

Проверьте, что индекс обновлён после последних изменений в Planfix. Затем
нажмите `Обновить индекс` и повторите поиск.

### Изменился токен Planfix

Замените токен в блоке `Подключение` и нажмите `Сохранить настройки`.

## Примечания для администратора Planfix

- приложение использует REST API Planfix только на чтение;
- токен должен иметь права на чтение нужных объектов, шаблонов задач,
  пользовательских полей, справочников и записей справочников;
- приложение не публикует токен в файлы выгрузок и не показывает его в ошибках.
