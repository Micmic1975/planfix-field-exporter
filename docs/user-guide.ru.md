# Руководство пользователя

## Для чего нужна программа

`Planfix Field Exporter` выгружает сведения о пользовательских полях Planfix в CSV-файл.

Программа поддерживает два источника:

- объект Planfix;
- шаблон задачи Planfix.

На выходе формируется CSV с техническими сведениями о полях: ID, тип, формула, параметры справочников, значения списков и полный JSON поля.

## Что нужно до начала работы

Перед первым запуском подготовьте:

1. имя аккаунта Planfix без суффикса `.planfix.ru`;
2. токен доступа REST API Planfix.

Пример:

- если адрес аккаунта:
  `https://engineering.planfix.ru`,
- то в поле **Аккаунт Planfix** нужно вводить:
  `engineering`.

Токен вводится без слова `Bearer`.

## Первый запуск

1. Запустите программу.
2. В блоке **Настройки** заполните:
   - **Аккаунт Planfix**;
   - **Токен доступа**.
3. При необходимости измените **Папку выгрузок**.
4. Нажмите **Сохранить настройки**.

После сохранения:

- аккаунт сохраняется в пользовательских настройках приложения;
- токен сохраняется в системном хранилище секретов Windows;
- программа проверяет, что аккаунт и токен подходят для обращения к Planfix;
- повторно вводить их при каждом запуске не требуется.

## Выгрузка полей объекта

1. В блоке **Что выгрузить** выберите **Поля объекта**.
2. В блоке **Выбор источника** выберите нужный объект.
3. При необходимости воспользуйтесь полем **Поиск**:
   - можно искать по части названия;
   - можно искать по ID.
4. Нажмите **Выгрузить**.

## Выгрузка полей шаблона задачи

1. В блоке **Что выгрузить** выберите **Поля шаблона задачи**.
2. В блоке **Выбор источника** выберите нужный шаблон.
3. При необходимости воспользуйтесь полем **Поиск**:
   - можно искать по части названия;
   - можно искать по ID.
4. Нажмите **Выгрузить**.

## Куда сохраняются файлы

По умолчанию CSV-файлы сохраняются в папку:

```text
Документы\Planfix Field Exporter
```

Папку можно изменить в блоке **Настройки**:

1. нажмите **Выбрать...**;
2. выберите нужный каталог;
3. нажмите **Сохранить настройки**.

Чтобы быстро открыть текущую папку выгрузок, нажмите **Открыть** рядом с полем папки.

Имя файла содержит:

- тип источника;
- ID источника;
- название объекта или шаблона.

Пример:

```text
planfix_task_template_82359_Описание курса _ для потенциальных клиентов_fields.csv
```

## Что содержится в CSV

Выгрузка содержит, в частности:

- тип и ID источника;
- название источника;
- ID поля;
- название поля;
- тип поля;
- формулу;
- параметры справочника;
- варианты списка;
- полный JSON поля.

CSV создаётся в кодировке, удобной для открытия в Microsoft Excel.

## Типовые ситуации

### Файл не сохраняется

Если программа пишет, что CSV-файл не удалось сохранить, скорее всего, файл с тем же именем уже открыт в Excel.

Что сделать:

1. закройте этот CSV в Excel;
2. повторите выгрузку.

### Нужный шаблон или объект не находится сразу

Используйте поле **Поиск**:

- введите часть названия;
- либо ID источника.

Программа фильтрует список сразу по мере ввода.

### Изменился токен доступа

1. В блоке **Настройки** замените токен;
2. нажмите **Сохранить настройки**.

Новые данные начнут использоваться сразу.

### Нужно открыть руководство

Нажмите кнопку **Руководство**. Откроется актуальная версия руководства в GitHub-репозитории проекта.

### Нужно узнать версию программы

Нажмите кнопку **О программе**. В окне будет указана версия приложения и ссылка на GitHub-репозиторий.

## Примечания для администратора Planfix

- Программа работает только на чтение: она получает данные через REST API и не изменяет объекты, шаблоны или поля в Planfix.
- Для корректной работы токен должен иметь права на чтение нужных сущностей.
- Списки объектов и шаблонов загружаются полностью, включая ситуации, когда их больше 100.
