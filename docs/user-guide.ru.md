# Руководство пользователя

## Для чего нужна программа

`Planfix Field Exporter` показывает сведения о пользовательских полях Planfix в таблице приложения.

Программа поддерживает два источника:

- объект Planfix;
- шаблон задачи Planfix.

На выходе отображается таблица с техническими сведениями о полях: ID, тип, формула, параметры справочников, значения списков и полный JSON поля.

## Что нужно до начала работы

Перед первым запуском подготовьте:

1. имя аккаунта Planfix без суффикса `.planfix.ru`;
2. токен доступа REST API Planfix.

Пример:

- если адрес аккаунта:
  `https://engineering.planfix.ru`,
- то в поле **Аккаунт Planfix** нужно вводить:
  `engineering`.

Токен вводится без слова `Bearer`.

## Первый запуск

1. Запустите программу.
2. В блоке **Подключение** заполните:
   - **Аккаунт Planfix**;
   - **Токен доступа**.
3. При необходимости измените **Папку выгрузок**.
4. Нажмите **Сохранить настройки**.

После сохранения:

- аккаунт сохраняется в пользовательских настройках приложения;
- токен сохраняется в системном хранилище секретов Windows;
- папка выгрузок сохраняется в настройках приложения;
- программа проверяет, что аккаунт и токен подходят для обращения к Planfix;
- повторно вводить их при каждом запуске не требуется.

### Предупреждение Windows SmartScreen

При первом запуске Windows может показать предупреждение SmartScreen:

```text
Система Windows защитила ваш компьютер
```

Это возможно потому, что приложение распространяется как небольшой самостоятельный `.exe` и пока не подписано коммерческим цифровым сертификатом издателя.

Если вы скачали файл из официального GitHub-релиза проекта, можно нажать **Подробнее** и затем **Выполнить в любом случае**.

Приложение не изменяет данные в Planfix и выполняет только запросы на чтение.

## Просмотр полей объекта

1. В блоке **Источник** выберите **Поля объекта**.
2. В блоке **Выбор источника** выберите нужный объект.
3. При необходимости воспользуйтесь полем **Поиск**:
   - можно искать по части названия;
   - можно искать по ID.
4. В блоке **Столбцы** отметьте нужные столбцы.
5. Поля выбранного объекта появятся в таблице автоматически.

## Просмотр полей шаблона задачи

1. В блоке **Источник** выберите **Поля шаблона задачи**.
2. В блоке **Выбор источника** выберите нужный шаблон.
3. При необходимости воспользуйтесь полем **Поиск**:
   - можно искать по части названия;
   - можно искать по ID.
4. В блоке **Столбцы** отметьте нужные столбцы.
5. Поля выбранного шаблона появятся в таблице автоматически.

## Выбор столбцов

В блоке **Столбцы** можно выбрать, какие данные показывать в таблице.

Доступны быстрые действия:

- **Основные** — показывает ID поля, название поля и тип поля;
- **Все** — включает все доступные столбцы;
- **Снять** — убирает все отметки.

Если столбцы не выбраны, программа попросит выбрать хотя бы один столбец.

## Что содержится в таблице

Таблица содержит, в частности:

- тип и ID источника;
- название источника;
- ID поля;
- название поля;
- тип поля;
- формулу;
- параметры справочника;
- варианты списка;
- полный JSON поля.

## Фильтрация и сортировка таблицы

В заголовках видимых столбцов есть поля фильтрации с иконкой воронки. Введите текст в поле нужного столбца, чтобы оставить строки, где этот столбец содержит введённое значение.

Если заполнить фильтры в нескольких столбцах, таблица покажет строки, которые подходят под все заполненные фильтры. Кнопка **Сбросить фильтры** очищает все фильтры таблицы.

Для сортировки нажмите заголовок нужного столбца со значком **↕**. Повторное нажатие меняет направление сортировки.

## Выгрузка в CSV

Кнопка **Экспорт CSV** сохраняет загруженные поля в выбранную папку выгрузок.

В CSV попадают:

- все загруженные строки выбранного объекта или шаблона;
- только отмеченные в блоке **Столбцы** столбцы.

Фильтры и сортировка таблицы на CSV не влияют.

## Копирование и контекстное меню

В таблице можно выбрать одно значение:

- щёлкните по нужной ячейке таблицы;
- **Ctrl+A** выделяет текущее значение таблицы;
- **Ctrl+C** копирует выбранное значение в буфер обмена.

Горячие клавиши работают независимо от текущей раскладки клавиатуры.

По правой кнопке мыши доступно контекстное меню:

- **Копировать значение** — копирует выбранную ячейку;
- **Фильтровать по значению** — подставляет выбранное значение в фильтр соответствующего столбца.

## Типовые ситуации

### Нужный шаблон или объект не находится сразу

Используйте поле **Поиск**:

- введите часть названия;
- либо ID источника.

Программа фильтрует список сразу по мере ввода.

### Изменился токен доступа

1. В блоке **Подключение** замените токен;
2. нажмите **Сохранить настройки**.

Новые данные начнут использоваться сразу.

### Нужно открыть руководство

Нажмите кнопку **Руководство**. Откроется актуальная версия руководства в GitHub-репозитории проекта.

### Нужно узнать версию программы

Нажмите кнопку **О программе**. В окне будет указана версия приложения и ссылка на GitHub-репозиторий.

## Примечания для администратора Planfix

- Программа работает только на чтение: она получает данные через REST API и не изменяет объекты, шаблоны или поля в Planfix.
- Для корректной работы токен должен иметь права на чтение нужных сущностей.
- Списки объектов и шаблонов загружаются полностью, включая ситуации, когда их больше 100.
